# CS5281 Internet Application Development (Sem B 2025–26) Assignment

**Client-side / server-side programming with HTML, CSS, JavaScript and PHP**

## Goal

Design and implement a web-based application system using **HTML**, **CSS**, **JavaScript**, and **PHP**.

## Group size

Approximately **8 students** per group (marking criteria are the same regardless of group size).

## Assignment description

- Students **can** propose their own topic—for example an online restaurant/hotel booking system, online survey, online quiz, online bidding system, etc. (You may refer to the appendix for topics from past years.)
- The client side is constructed with **HTML** and **CSS**, using **JavaScript** as the scripting language.
- Before information is sent from client to server, **JavaScript** should be used to check validity of data (e.g. invalid characters or quantities, incorrect dates, etc.).
- Beyond validation, **JavaScript** may be used for animation, statistics, time management, message processing, shopping cart or cookie management, etc.
- The server side should be implemented with **PHP** (frameworks optional if you prefer). Scalability may be ignored; it is acceptable to store information directly in text or binary files instead of using a database.
- Students might also use **SQL** if they prefer. However, in order to be fair to those without database background, there will be no special support from tutor and no special bonus favoring students who use SQL.
- **PHP** scripts should use form data, cookies/sessions, text/binary files, and a reasonable amount of array and string operations.
- The system should work in mainstream browsers such as **Safari**, **Chrome**, and **Firefox**.

## Hosting and submission

- Students may host the web system using:
  1. Personal notebook, probably with Apache/PHP/database (or package like XAMPP) installed (e.g. [https://www.apachefriends.org/](https://www.apachefriends.org/)).
  2. Online free web hosting like 000webhost or InfinityFree ([https://www.000webhost.com/](https://www.000webhost.com/), [https://infinityfree.net/](https://infinityfree.net/)).
- Students also have to submit the source files (PHP, HTML, CSS, …etc.) as a **single zip file** to **Canvas** on **Week 13**.
- Bonus marks would be given if advanced techniques like **AJAX** are used whenever appropriate.
- **Report is optional.** Students may include it for bonus (e.g. a short one within 10 pages to briefly describe the design, architecture, and functions with screen captures). There is **NO** need to consider business-side issues like SWOT analysis, revenue, customer, competitor…etc.

### Effort (25 marks)

- Does the system include all necessary/reasonable features? (e.g. registration, report generation, confirmation email)
- *A system with about 5 features should be good enough.*

### Design (10 marks)

- Is the system realistic?
- Is the system efficient?
- Is the system secure / fool-proof?

### Correctness (12 marks)

- Can the system handle incorrect input?
- Are the assumptions valid?
- Any errors or exceptions during demonstration?

### HTML skill (10 marks)

- Form design
- Syntax conformance (works in *any* browser?)

### CSS skill (8 marks)

- Sufficient use of CSS?
- Consideration of media?
- High usability? (e.g. avoiding black text on a dark background)

### Use of JavaScript (15 marks)

- e.g. form data verification
- e.g. shopping cart
- e.g. other features (animation, popup messages, etc.)

### Use of PHP (20 marks)

- e.g. File / Server-side information management
- e.g. Use of session or cookie if necessary
